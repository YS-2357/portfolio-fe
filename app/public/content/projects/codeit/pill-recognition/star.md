## 🔹 S (Situation) — 프로젝트 배경

> 환자 및 의료진은 육안으로 유사한 알약을 구분하기 어려우며, 이는 복약 오류와 약물 사고로 이어질 수 있습니다. 
이러한 문제를 해결하고자 헬스케어 분야에 객체 탐지(Object Detection) 기술을 활용하여, 사용자가 촬영한 알약 이미지를 분석해 자동으로 알약의 위치(Bounding Box)와 종류(Class)를 인식하는 AI 모델 개발이 필요했습니다.
> 

---

## 🔹 T (Task) — 내가 맡은 역할과 과제

> 팀장으로서 프로젝트의 전반적인 방향과 일정 조율을 담당했으며, Faster R-CNN과 YOLO 기반 객체 탐지 실험을 설계·주도했습니다. 
Git 기반 협업 구조를 제안하고, 전체 파이프라인과 CLI 실행 구조를 통합하여 자동화와 실험 재현성을 확보했습니다.
> 

---

## 🔹 A (Action) — 핵심 실행 작업

### 🧰 구조 설계 및 자동화

- Git 브랜치 전략 수립 (Feature → Develop → Main)
- `main.py` 중심의 CLI 기반 단일 실행 구조 구축
- 학습 → 검증 → 예측 → 제출까지 파이프라인 자동화
- 실험 결과, 파라미터, 성능 기록을 위한 노션 기반 실험 관리

### 📊 데이터 정비 및 파이프라인 구성

- 총 1,489개 이미지와 어노테이션 중 26건 이상 오류 수작업 보완, 데이터셋 신뢰성 확보
- COCO 포맷 정리 및 알약 category-mapping 통일
- 데이터 로더/전처리/변환 모듈 설계 및 구현

### 🧪 객체 탐지 모델 실험 설계

- **Faster R-CNN (MobileNet_v3)**
    - Loss classifier 조정, Focal Loss 적용, Backbone 성능 비교
    - mAP@0.5 = 0.8752, Kaggle score = 0.39573
- **YOLOv8**
    - 학습, 검증, 시험의 파이프라인 및 시각화 전체 구조 설계 및 구현
    - 하이퍼파라미터 비교 실험 및 로깅
    - mAP@0.5 = 0.99334, Kaggle score = 0.99857

## 📌 회고 (Reflection)

- **데이터 전처리가 가장 중요함**을 깨달았습니다. 양질의 데이터가 모델의 성능을 좌지우지함을 느꼈습니다.
- **모델 성능뿐만 아니라 파이프라인의 자동화, 실험 재현성, 협업 구조가 실질적인 성과를 결정짓는 요소**임을 체감하였습니다.
- 객체 탐지 모델 선택은 **정확도**뿐만 아니라 **추론 속도, 경량성**까지 고려해야 한다는 것을 깨달았습니다.
- YOLOv8의 강력한 성능에도 불구하고, Faster R-CNN을 통해 얻은 분석적 통찰이 분류 문제 이해에 큰 도움이 되었고, 두 모델 모두 실험적으로 균형 있게 접근하였습니다.
- 프로젝트를 진행하며 가장 중요했던 점은 **팀원 간의 신뢰와 분업 구조**였습니다. 역할을 명확히 나누고, 핵심 파트마다 리더 역할을 수행하면서 전체 흐름을 정리하고 끌어가는 경험을 할 수 있었습니다.
- 프로젝트를 마무리하며 개인 포트폴리오에 해당 프로젝트를 담는 작업을 진행하였고, **매일의 협업일지와 코드 기록을 통해 자연스럽게 하나의 문서화 흐름**을 형성할 수 있었습니다.
- 이를 통해 **기술적 성과 외에도 협업 경험을 스스로 정리하며 성찰하는 과정**이 되었습니다.
