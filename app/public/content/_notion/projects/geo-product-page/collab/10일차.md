### [2] 오늘 맡은 역할 및 구체적인 작업 내용


✍️ 답변:


```plain text
1. FastAPI 라우터를 통해 이미지 생성기를 실행할 수 있도록 전체 API 연결 작업을 수행  
	- `image_router` 모듈을 `main.py`에 등록하여 `/api/image/create` 라우트를 정상적으로 작동하게 함

	- 입력된 제품 이미지 경로를 기반으로 이미지 로딩, 배경 제거, 프롬프트 생성, 모델 로딩, IPAdapter 및 Inpainting 기반 이미지 생성까지 실행됨을 확인  
	- 로그 기반 확인 및 결과 이미지 경로 반환을 통해 Streamlit에서 감지 가능한 구조로 마무리  
	- 날짜 기반 시드를 적용하여 생성 결과의 랜덤성을 확보하도록 구조화
2. 허깅페이스 블로그 기반 Virtual Try-On using IP-Adapter Inpainting 방법 확인
	- 수동 마스킹을 적용한 경우 제품 디테일이 보존되면서 입혀지는 결과를 확인했고, 이를 토대로 향후 자동 마스킹 전략을 구상함  
	- 제품 이미지의 디테일 보존을 위해 IPAdapter scale, strength, guidance_scale 등을 조정하는 실험
	- IPAdapter, SDXL Inpainting, LoRA(weight: sd-tryon-model-lora-sdxl) 등을 조합한 가변 파이프라인을 실험  
```

### [5] 오늘 분석/실험 중 얻은 인사이트나 발견한 문제점은?


✍️ 답변:


```plain text
- IPAdapter는 "디테일 복사"가 아닌 "스타일 기반 조건 부여"이기 때문에 제품 이미지의 로고, 무늬, 질감 등이 일정 수준 왜곡될 수 있음  
- diffusion 모델의 경우 `prompt`에 과한 정보가 들어가면 오히려 디테일을 망치는 경우가 있어 prompt는 최소화해야 함  
- `torch.manual_seed()`를 이용한 시각 기반 랜덤 시드가 안정적인 실험 reproducibility를 보장함  
```

