### [5] 오늘 분석/실험 중 얻은 인사이트나 발견한 문제점은?


✍️ 답변:


```plain text
# DALL·E
- 무료 API 사용 가능하지만 커스터마이징 및 이미지 입력 불가

# StabilityAI
- 텍스트의 경우 API 무료 횟수를 소모, 이후 허깅페이스 모델로 fallback 고려
- API는 텍스트 기반만 가능 → 이미지 입력을 위해선 Hugging Face diffusers 사용
- 비동기 요청 (httpx/asyncio) 기반 구성 필요
- 기본 requests(동기)보다 메모리 효율성 확보 가능
- aiohttp도 고려 가능하지만 비동기만 처리함. 개발편의를 위해서는 httpx(동기/비동기)가 더 좋음

# ControlNet 기반 이미지 입력 가능 모델 존재
- Hugging Face diffusers 문서 참고: StableDiffusionImg2ImgPipeline, ControlNetPipeline

아래 폴더 구조에서 상대경로 import 문제 발생

2025-GEO-Project/
├── backend/
│   └── image_generator/
│       ├── api/
│       ├── core/
│       ├── schemas/
│       └── image_generator_main.py
├── utils/
│   └── logger.py

해결 방법:
- `.env`에 PYTHONPATH=/home/user/2025-GEO-Project 추가
- sys.path.append(os.getenv("PYTHONPATH")) 설정
- import 전에 반드시 적용
```
