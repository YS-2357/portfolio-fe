# **8. 📈 개발 이슈 및 해결 방안**


> ⚙️ 


**문제 정의 및 폴더 구조 정리**


- **문제**:  ‘소상공인을 위한 상세페이지 개발’이라는 추상적인 명제의 과제 목표로 혼란을 겪음

- **해결**: 팀원 전부 각자의 아이디어를 합쳐 회의를 통해 정리



**로깅에 대한 기준 부재**


- **문제**: 어느 부분을 어떻게 로깅을 할 지에 대한 기준이 없음

- **해결**: logging 라이브러리를 사용, 표준이 되는 약속을 정리



**VRAM 초과로 인해 서버 다운**


- **문제**: 허깅페이스 모델 다운로드 및 로딩 과정 중 VRAM초과로 인해  GCP 서버 다운

- **해결**: VRAM을 늘릴 수 없어 서버 재시작으로 해결



**DB 접근**


- **문제**: GCP 재부팅 시 IP 변경 → config.yaml 수정 및 커밋이 번거로움

- **해결**: DB IP를 .env에서 불러오도록 바꿔서 깃 커밋 없이 유연하게 관리



> 📂 


**입력 데이터 유효성 문제**


- **문제**: 프론트에서 보내는 데이터가 예상한 스키마와 다를 경우, Pydantic이나 내부 검증에서 오류 발생

- **해결**: FormParser에서 강력한 정규화 & 예외 처리 필요



**이미지 경로 처리 오류**


- **문제**: 로컬 경로/URL 경로 구분이 명확하지 않거나(절대 경로, 상대경로), 파일이 존재하지 않을 경우 처리 중단

- **해결**: 파일 유효성 검사 로직을 ImagePreprocessor와 InputHandler 양쪽에 중복으로 두는 대신 **중앙 처리 유틸 함수**로 통일할 것



**config.yaml 저장 실패 또는 구조 오류**


- **문제**: 저장 중 인코딩 문제, 잘못된 데이터 타입 포함 시 YAML 파일 포맷이 깨지거나 저장 실패

- **해결 **: `Dict[str, Any]` 구조 정합성 검증 후 `yaml.safe_dump(config, allow_unicode=True)`로 저장 처리



> 🌐 


**사이트의 봇 접근 차단**


- **문제**: 크롤링 대상 사이트가 봇 의심 접근을 차단하여 페이지에 접근할 수 없음

- **해결**: Selenium으로 브라우저를 직접 실행하고, fake_useragent를 사용해 일반 사용자처럼 접근



**사이트의 봇 의심 IP 차단**


- **문제**: 과도한 요청으로 IP가 임시 차단됨

- **해결**: VPN을 사용해 IP를 우회하고, 차단 시 다른 IP로 교체하여 크롤링 지속



**GCP 환경 내 VPN 사용의 제약**


- **문제**: GCP에서 VPN을 실행하면 IP가 변경되어 SSH 접속이 끊기거나 관리가 어려워짐

- **해결**: 크롤링은 local 환경에서 수행하고, 결과만 GCP로 연동하는 구조로 분리



**local 환경과 GCP 간 데이터 연동 문제**


- **문제**: local에서 크롤링한 데이터를 GCP에 어떻게 전달할지 불명확

- **해결**: GCP에 MySQL DB를 설치하고, local 환경에서 직접 DB에 접근하여 데이터 저장



**정적 리뷰 데이터로 인한 최신성 저하**


- **문제**: 리뷰 요약본을 DB에 저장해 사용하다 보니 최신 리뷰가 반영되지 않는 문제 발생

- **해결**: 저장된 상품 카테고리들을 주기적으로 다시 크롤링하여 최신 리뷰로 갱신



**사용자 요청 처리 로직 부재**


- **문제**: DB에 없는 카테고리를 사용자가 요청하면 처리 불가

- **해결**: 신호 테이블을 통해 GCP가 요청 신호를 보내면 local에서 감지 후 크롤링 수행



**DB에 데이터가 계속 축적됨**


- **문제**: `INSERT` 방식으로 저장하여 중복 데이터가 누적됨

- **해결**: `UPSERT` 쿼리(`ON DUPLICATE KEY UPDATE`)를 사용하여 기존 카테고리는 덮어쓰도록 변경



**local 실행 시 크롤링이 무조건 실행됨**


- **문제**: local 코드를 실행할 때마다 즉시 크롤링이 수행되어 주기 조건과 무관하게 동작함

- **해결**: DB에 저장된 마지막 크롤링 시각을 참조하여, 주기가 지난 경우에만 크롤링 수행



> 🖼️ 


**모델 다운로드 실패 (디스크 부족)**


- **문제**: stabilityai/stable-diffusion-3.5-medium 다운로드 중 No space left on device에러

- **해결**: 불필요한 데이터 삭제로 30GB확보 후, 모델 변경(SDXL 기반 경량 모델)



**Stable Diffusion 2.1 사용시 의류 합성 실패**


- **문제**: 상품 이미지 기반 의류 착용 합성 불가능 → 인체 포즈 반영 안 됨

- **해결**: ControlNet(OpenPose) 적용 및 rembg로 배경 제거 후 재시도



**생성된 이미지 내 상품 디테일 왜곡**


- **문제**: SD 2.1에서 원본 상품 텍스처가 심하게 변형됨

- **해결**: IP-Adapter 기반 구조로 변경, 상품 특징 주입 방식 강화



**한글 프롬프트 비호환**


- **문제**: Stable Diffusion 모델이 한국어 프롬프트에서 품질 저하

- **해결**: 영어 기반 프롬프트로 변경, 번역 + 자연스러운 문장 구조 적용



**배경 복잡도 및 장면 중복**


- **문제**: 프롬프트 적용 시 배경이 복잡하고, 이미지에 2개의 장면이 합성되는 현상 발생

- **해결**: 프롬프트에 단일 scene 지시어 및 배경 단순화 지침 추가 → 개선 확인



**부정 프롬프트 효과 미비**


- **문제**: Negative Prompt 추가 후에도 생성 이미지의 차이가 미미

- **해결**: Negative Prompt 항목을 세분화(색상, 오브젝트, 로고 등)



**rembg 배경 제거 품질 불균일**


- **문제**: rembg로 배경 제거 시, 경계선이 거칠고 노이즈 발생

- **해결**: `alpha_matting` 활성화 + Threshold 조정 → 품질 개선



**프롬프트 생성기 품질 불안정**


- **문제**: 프롬프트 자동 생성 시 배경 설정이 지나치게 복잡하거나 어색함

- **해결**: 생성 프롬프트 길이를 15단어 이내로 단순화, 핵심 지시어 유지





**모델별 품질 차이 및 리소스 문제**


- **문제**: SD 3.5 품질 우수, 로드 시간·용량 부담 큼 / SDXL Turbo 속도 빠름, 일부 디테일 부족

- **해결**: 최종적으로 SDXL Turbo 채택 (속도·품질 균형)



**ControlNet 단독 성능 한계**


- **문제**: OpenPose/Canny 적용 후에도 자연스러운 합성이 어려움

- **해결**: ControlNet + IP-Adapter 병행 가능성 탐색, 스타일과 포즈 동시 반영



**SDXL Turbo 및 SDXL Base 이미지 품질 저조**


- **문제**: 배경 합성은 가능하지만 의류 착용 구현 불가능

- **해결**: SDXL Turbo를 상품 배경 생성 용도로만 활용, 사람+의류 합성은 인페인팅 기반으로 전환



**마스크 기반 인페인팅 시도**


- **문제**: Stable Diffusion 2 Inpainting 사용 시 테두리 불명확, 자연스러움 부족

- **해결**: 마스크 정교화 전략 검토 → 프로토타입 수준에서 제한적 활용



**이미지 생성 전략 재정립**


- **문제**: 완전한 VTON 구현은 시간·리소스 한계로 어려움

- **해결**: **단기 전략**: IP-Adapter + SDXL 조합으로 프로토타입 완성
          **장기 전략**: IDM-VTON 또는 TryOnDiffusion 활용, ControlNet과 FaceID 연동 고려



**IP-Adapter 기반 사람 이미지 품질 불안정**


- **문제**: 손가락 등 일부 디테일 비현실적

- **해결**: 추론 횟수 15→50회로 증가 및 이미지 크기 768×768로 조정



**IP-Adapter + SDXL 사용 시 상품 강조 부족**


- **문제**: 배경에 비해 상품이 잘 부각되지 않음

- **해결**: 프롬프트에 `photorealistic`, `product-focused`, `studio lighting`등 구체 키워드 삽입



**VTON 사용 시 마스킹 적용 문제**


- **문제**: 정밀 마스킹 자동화 어려움 → 일부 이미지에서 합성 품질 저하

- **해결**: 초기에는 수동 마스크 적용, 추후 자동화 기능 탐색 예정



**이미지 객체 반환 오류**


- **문제**: FastAPI 응답 시 `PIL.Image` 객체는 JSON 직렬화 불가 → API 응답 실패

- **해결**: 이미지 경로(path)를 반환하는 방식으로 수정



**시드 값 고정으로 인한 이미지 다양성 부족**


- **문제**: 동일한 시드 값으로 매번 유사 이미지 생성 → 결과물이 반복적

- **해결**: 날짜 기반 랜덤 시드(년월일시분초) 적용



**IP-Adapter Virtual Try-On 정확도 부족**


- **문제**: 초기 시도 시 의류 위치가 어색하고 사람 이미지 품질 낮음

- **해결**: HuggingFace 블로그 기반 파이프라인 재구성 및 파라미터 조정으로 개선



**원본 의류 영향으로 합성 불안정**


- **문제**: 기존 옷이 남아있으면 새로운 옷 합성 자연스럽지 않음

- **해결**: 옅은 의상 착용 이미지 사용 + 마스킹으로 타겟 구역 분리



**마스킹 작업 자동화 미비**


- **문제**: 상의나 특정 부위만 정확히 마스킹하는 자동화 기능 부재 → 반복 작업 비효율

- **해결**: 사용자가 인물 이미지와 마스킹 이미지를 선택하는 방식으로 대응



**이미지 생성 속도 문제**


- **문제**: IP-Adapter + ControlNet + Inpainting + LoRA 조합 시 평균 생성 시간 2분 30초 소요

- **해결**: 모델 최초 1회 로드, 추론 횟수 감소로 30초 이내로 단축



**인물 이미지 자세에 따른 품질 편차**


- **문제**: 모델 자세가 비대칭이면 합성 왜곡 발생

- **해결**: 정자세 모델 이미지를 선별해 적용



**VTON 자체의 이미지 품질 낮음**


- **문제**: Virtual Try-On 시 인체 비율 및 상품 디테일 손상

- **해결**: VTON API 활용으로 정확도 높은 결과 확보



> ⌨️ 


**HuggingFace 모델에 따른 GPU VRAM 부족**


- **문제**: 허깅페이스 모델의 크기로 인해 모델의 추론 속도 지연 및 메모리 초과 발생

- **해결**: 모델 양자화를 적용해 메모리 사용량을 줄이고 다양한 경량 모델을 실험하여 최적화



**Hugging Face 모델의 상세페이지 생성 성능 부족**


- **문제**: Gukbap-Qwen2.5-7B 모델은 GEO 최적화 지시를 완전히 반영하지 못해 상세페이지 품질이 낮음

- **해결**: OpenAI 생성 결과 50개를 학습 데이터로 활용하여 LoRA 방식으로 추가 학습을 진행

- **효과**: 문장 유창성이 향상되었으며, 전문가 인용 파트가 생기는 등 GEO 최적화 지시를 반영



**OpenAI 생성 결과의 시각적 품질 문제**


- **문제**: 기존에는 HTML 텍스트만 생성하고, 사전 제작한 CSS 템플릿을 병합하는 방식이었으나, 실제 온라인 판매 환경에서 사용하기에는 디자인 품질이 낮음

- **해결**: 프롬프트에 Tailwind CSS를 사용하도록 명시하여 OpenAI가 HTML + CSS 전체를 완성도 높게 생성하도록 변경

- **효과**: 실제 상품 상세페이지와 유사한 결과물로 시각적 완성도 확보



**CSS 디자인의 일관성 문제**


- **문제**: OpenAI가 생성하는 CSS가 요청할 때마다 달라져 디자인 일관성 저하

- **해결**: 특정 HTML 태그에 적용할 CSS 클래스를 프롬프트에 지정 (예: 고객 정보는<p class="font-semibold text-blue-700">로 표시할 것)



**이미지 태그의 AI 검색 노출 최적화**


- **문제**: 기존 HTML 내 이미지 태그에는 alt 속성이 없어 AI 검색 노출 최적화가 부족

- **해결**: 상품 특징을 바탕으로 alt 속성을 생성하도록 프롬프트를 수정

- **해결**: setup.sh 스크립트를 구성하여 apt, pip, playwright install 명령어를 일괄 실행 가능하도록 자동화
