## 🔧 프로젝트 1: insure-gpt (백엔드)

### 작업 1: Swagger UI 문서 대폭 개선

#### 🤔 Swagger UI가 뭔가요?

Swagger UI는 API(Application Programming Interface)를 테스트하고 문서를 볼 수 있는 웹 페이지입니다.

- **API란?** 프로그램끼리 대화하는 방법입니다. 예를 들어, 프론트엔드(웹사이트)가 백엔드(서버)에 "이 PDF 분석해줘"라고 요청하면, 백엔드가 분석 결과를 돌려주는 것이 API입니다.
- **Swagger UI 접속 방법:** 서버 실행 후 `http://localhost:8000/docs`로 접속

#### ✅ 변경 내용

**변경 전 문제점:**
- API 설명이 너무 간단해서 처음 보는 사람이 이해하기 어려웠음
- 에러가 발생했을 때 어떻게 처리해야 하는지 안내가 없었음

**변경 후 개선점:**

1. **API 사용법을 단계별로 설명 추가**
   ```
   1단계: 요청 보내기 (어떤 데이터를 어떻게 보내는지)
   2단계: 응답 받기 (어떤 데이터가 돌아오는지)
   3단계: 에러 처리 (문제가 생겼을 때 어떻게 하는지)
   ```

2. **실제 코드 예시 추가**
   - curl (터미널에서 API 호출하는 명령어)
   - Python (파이썬 코드 예시)
   - JavaScript (자바스크립트 코드 예시)

3. **에러 응답 상세 설명 추가**
   - HTTP 400: 잘못된 요청 (예: PDF 파일이 아닌 파일 업로드)
   - HTTP 422: 데이터 검증 실패 (예: 필수 필드 누락)
   - HTTP 500: 서버 내부 오류 (예: AI 모델 호출 실패)

4. **각 필드(변수)별 상세 설명 추가**
   - 요청할 때 보내는 데이터가 무엇인지
   - 응답으로 받는 데이터가 무엇인지

#### 📁 변경된 파일

| 파일 | 설명 |
|------|------|
| `backend/app.py` | FastAPI 서버 메인 파일 (API 문서 내용이 여기에 있음) |

### 작업 3: 로그 레벨 변경 (디버깅용)

#### 🤔 로그가 뭔가요?

로그(Log)는 프로그램이 실행되면서 남기는 기록입니다. 마치 일기장처럼, "지금 이 작업을 하고 있어요", "여기서 문제가 생겼어요"를 기록해둡니다.

#### ✅ 변경 내용

`logger.debug` → `logger.info`로 변경

- **debug**: 개발자만 보는 상세한 로그 (기본 설정에서 안 보임)
- **info**: 일반적인 정보 로그 (기본 설정에서 보임)

이 변경으로 보장명 매칭 과정을 터미널에서 바로 확인할 수 있게 되었습니다.

## 🎨 프로젝트 2: partners-care-old (프론트엔드)

### 작업: 보험 분석 결과 UI 개선

#### 🤔 무엇을 바꿨나요?

보험 분석 결과를 보여주는 화면을 더 보기 좋게 개선했습니다. 기존에는 단순하게 결과만 나열했는데, 이제는 탭으로 구분하고 시각적으로 예쁘게 표시합니다.

#### ✅ 변경 전 vs 변경 후

**변경 전:**
- 분석 결과가 한 페이지에 길게 나열됨
- 어떤 정보가 중요한지 구분이 어려움
- 스타일이 단순함

**변경 후:**
- **3개의 탭**으로 구분:
  - 📊 **요약 탭**: 핵심 정보만 빠르게 확인
  - 🔍 **개별 분석 탭**: 보장 항목별 상세 분석
  - 📝 **종합 분석 탭**: AI의 전체 의견과 영역별 요약

#### 📊 요약 탭에서 보여주는 것

| 구분 | 내용 |
|------|------|
| 고객 정보 | 나이대, 성별, 직업, 월소득 |
| 실손보험 분석 | 가입여부, 세대(1세대/2세대/4세대), 판별근거 |
| 보험료 분석 | 총 월납입료, 소득 대비 비율, 적정/주의/과다 판정 |
| 보장 분석 통계 | 평가한 보장 항목 수, 적정 비율 |
| 주요 이슈 | 부족한 보장, 과다한 보장 목록 |

#### 🔍 개별 분석 탭에서 보여주는 것

각 보장 영역(암, 뇌, 심장 등)을 **아코디언**(접었다 펼 수 있는 UI)으로 구분:

- 보장명 + 진단결과(적정/부족/과다/미가입)
- 중요도 배지 (필수/권장/선택/참고)
- 금액 비교: 고객 가입 vs 권장 범위 vs 차이
- 판단근거 설명
- 상세내역 (어떤 보험사, 어떤 상품인지)
- AI 분석 코멘트

#### 📝 종합 분석 탭에서 보여주는 것

- AI가 작성한 종합 의견
- 영역별 적정률 **프로그레스 바** (시각적으로 몇 %인지 표시)
- 주요 부족/과다 항목 요약

## 💡 오늘 배운 것 / 트러블슈팅

### 1. Hot Reload vs 서버 재시작

**Hot Reload란?**
코드를 수정하면 서버를 재시작하지 않아도 자동으로 반영되는 기능입니다.

| 프레임워크 | Hot Reload 지원 |
|------------|-----------------|
| Next.js (npm run dev) | ✅ 대부분 자동 반영 |
| FastAPI (uvicorn --reload) | ✅ 대부분 자동 반영 |

**단, 재시작이 필요한 경우:**
- 설정 파일 변경 (next.config.js, .env)
- 의존성 변경 (package.json, requirements.txt)
- 가끔 캐시 문제로 반영이 안 될 때

### 2. Next.js "Cannot read properties of null" 에러

**문제:** 브라우저에서 `removeChild` 에러 발생

**해결 방법:**
1. 브라우저 강력 새로고침: `Ctrl + Shift + R`
2. `.next` 폴더 삭제 후 재시작: `rm -rf .next && npm run dev`

### 3. Python 코드 수정이 반영 안 되는 경우

**원인:**
- 서버가 계속 실행 중이면 수정된 코드가 반영 안 됨
- `__pycache__` 폴더에 이전 컴파일된 코드가 남아있음

**해결:**
```bash
# 서버 완전히 중지 후 재시작
Ctrl+C
uvicorn backend.app:app --reload --port 8000
```
