# 프론트엔드 구조

## 목적
React + TypeScript 정적 포트폴리오 사이트와 플로팅 챗봇 UI를 제공한다.

## 콘텐츠 소유
- 이력서/포트폴리오 콘텐츠는 프론트엔드가 소스 오브 트루스.
- 콘텐츠 위치와 파일 구조를 고정해 백엔드가 참조할 수 있게 한다.

## 라우팅
- 라우트 목록과 목적을 여기에 정리한다.
- 챗봇이 연결해야 할 정식 URL을 명시한다.
- 현재 라우트:
  - `/`: 랜딩 페이지
  - `/awards`: 전체 수상 목록
  - `/projects/codeit/:project/:page`: 프로젝트 상세(`page`는 `star` 또는 `report`)

## 챗 위젯
- 화면 측면에 플로팅 UI로 고정.
- 답변에 관련 페이지 링크를 포함할 수 있어야 한다.
